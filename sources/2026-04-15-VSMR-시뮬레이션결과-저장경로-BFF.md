# VSMR 시뮬레이션 결과 저장 경로 (BFF 담당자 전달)

## 저장 경로 구조

```
v-smr/[사용자 uuid]/[project-id]/simulation/history/[simulation-id]/{task_index}/{출력파일}
```

## task_index 설명

- mars-adaptor에서는 model-id를 알 수 없어서 task_index를 사용
- task_index는 model의 순서와 동일 (0 = 첫번째 모델)
- restart 시 설정에 project_id, simulation_id, model_id를 입력받지만, 실제 사용 시에는 task_index가 필요
- task_index를 model의 순서로 사용하도록 수정 예정 (BFF 담당자)

## 최근 결과 저장 폐지

- 기존에 최근 결과를 별도 저장하던 동작을 더 이상 하지 않음
