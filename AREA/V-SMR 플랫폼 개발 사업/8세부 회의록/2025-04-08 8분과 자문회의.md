# WorkFlow에 관한 논의

#### Task Manager 
- input file path 전달 시 TaskManager로 전달되고
- 쿠버네티스 환경의 솔버로 정보를 전달하여 해석을 시작함

#### 스토리지 구조
버킷: vsmr 
	사용자: alice 
		프로젝트: projectName(프로젝트 생성시 입력한 이름 & 중복 체크)
				inputFile: mars_input_file.i
				resultFile: 해석 완료시 업로드 
				
### Sim Start
해석 요청 인자
- Bucket Name 
- User/Project/mars_input.i
### Sim Process
Start 요청 후, Editor 환경-> Sim Monitoring 환경으로 전환 필요
- React 라우팅 추가 고려
- plot data window
- log window 
- result navigation window (해석 결과 리스트업 및 다운로드)
#### Data Request Function 
- Screen과 Plot용 요청이 별도로 분리되어 있으며, 
- Plot의 경우 msec 까지 요청이 필요할 수  있음
### Sim End
- 해석이 끝나면 유니크한 시그널(eol)을 전달할 예정
- 이 시점에 결과파일 업로드는 완료된 상태.
