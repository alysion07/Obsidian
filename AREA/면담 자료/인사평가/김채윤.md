## 1. 수행 프로젝트 성과 심층 질문

평가표의 '실적 근거'에 기재된 기술적 기여도를 검증하고 고도화 가능성을 묻는 질문입니다.

### [NFLOW 3.0 (MFC) 기능 개편 및 사용성 개선]

- **구조적 설계:** "Solver 옵션과 Add-on 설버를 고려한 설정 구조를 설계하셨는데, 향후 새로운 물리 모델이 추가될 때 코드 수정 최소화를 위해 어떤 설계 패턴이나 원칙을 적용하셨나요?"
    
- **사용성 개선의 기준:** "UI 동작 재정의와 리소스 효율화 작업을 수행하셨습니다. 단순히 보기 좋아진 것 외에, 실제 작업자의 프로세스 단계를 줄이거나 속도를 개선한 수치적 사례가 있을까요?"
    
- **품질 관리:** "메모리 누수 및 UI 현상 유발 요인을 클리닝하셨습니다. 반복되는 버그를 방지하기 위해 별도의 디버깅 체크리스트나 가이드를 팀 내에 공유하신 적이 있나요?"
    

### [NFLOW Web 백엔드 및 TaskManager 구축]

- **시스템 안정성:** "다중 서버 Queue와 Task 할당 구조를 구축하셨습니다. 서버 부하가 급증하거나 특정 서버에 장애가 발생했을 때, 시스템의 가용성을 보장하기 위해 어떤 예외 처리 로직을 구현하셨나요?"
    
- **기술 전환의 어려움:** "데스크톱(MFC) 환경과 달리 웹 환경에서의 시뮬레이션 모니터링은 리소스 제약이 큽니다. 실시간 상태 확인 기능을 구현하면서 가장 고민했던 성능 최적화 포인트는 무엇이었나요?"

## 2. 역량평가 항목별 보완 질문

1차 평가 점수가 80~85점인 항목들을 중심으로, 평가자가 더 높은 점수를 줄 수 있도록 근거를 이끌어내는 질문입니다.

### [전문성: 해결도(80점) / 숙련도(85점)]

- "업무 수행 중 기술적으로 가장 해결하기 어려웠던 병목 현상은 무엇이었으며, 이를 해결하기 위해 기존 방식이 아닌 새로운 접근법을 시도했던 사례가 있다면 설명해 주세요."
    

### [몰입도: 주도성(80점) / 집중도(85점)]

- "백엔드 시스템 구축 시 기획서에 명시되지 않았지만, 실제 운영 효율을 위해 본인이 선제적으로 제안하여 반영된 기능이나 로직이 있습니까?"
    
- "여러 프로젝트(MFC, Web)를 동시에 수행하면서 업무의 우선순위를 어떻게 관리하셨는지, 몰입을 방해하는 요소를 어떻게 제어하셨는지 궁금합니다."
    

### [조직성: 협업(85점)]

- "전처리 스크립트 작성 및 데이터 파싱 작업은 다른 파트와의 데이터 정합성이 중요합니다. 타 개발자와의 인터페이스 협의 과정에서 발생한 의견 차이를 논리적으로 설득하여 해결한 경험이 있나요?"

## 💡 용어 설명

- **Add-on Solver:** 기본 해석 프로그램 외에 특정 물리 현상이나 기능을 추가로 계산하기 위해 붙이는 보조 연산 모듈입니다.
    
- **메모리 누수 (Memory Leak):** 프로그램이 필요하지 않은 메모리를 계속 점유하여 시스템 성능을 저하시키는 현상입니다.
    
- **Queue (큐):** 데이터를 일시적으로 저장하는 자료구조로, 백엔드에서 작업(Task)을 순차적으로 처리하거나 서버 부하를 분산할 때 사용합니다.
    
- **토큰화 (Tokenization):** 데이터를 유의미한 최소 단위(토큰)로 나누는 과정으로, 여기서는 시뮬레이션 상태 데이터를 효율적으로 처리하기 위한 전처리를 의미합니다.
