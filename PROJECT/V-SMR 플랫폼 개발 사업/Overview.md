## 🔹 1. KAERI 내 원전 시뮬레이터 소개
### ● Compact Nuclear Simulator (CNS)
- 병렬 처리 및 Shared memory 기반 실시간 제어 가능 → 강화학습에 적합.
- KAERI 보유 중이나 SMR 과제에는 적합하지 않음 (iPWR 사용 중).
- 상업적 활용은 불명확, 모델 참고용으로 활용 가능.
### ● 3KEYMASTER Simulator (Western Services Corp.)
- APR-1400 등 대형 원전 기반, 가속화는 어려우나 사용자 편의성과 데이터 저장 우수.
- 자율운전 모델 개발에는 적합하지 않음.
### ● iPWR (Integral Pressurized Water Reactor)
- Tecnatom 제공 간이형 시뮬레이터, 일부 보조계통 미구현, 열수력 모델 단순.
- SMR 과제에 적합하여 현재 개발 중, ENU와 API 및 인터페이스 공동 개발 중.
### ● 논의 사항
- 열수력 코드 기반 시뮬레이터는 상대적으로 단순함 (노심, BOP 미구현).
- 핵연 제어 통한 출력 변동 시뮬레이션 가능, MARS 코드 활용 예정.

## 🔹 2. 열수력 코드 (MARS) 관련

### ● 현황 및 한계
- ETRI 보유 MARS는 실시간 제어 불가, 사전 정의된 입력만 처리 가능.
- CNS와 달리 Shared memory 제어 불가, 동적 변수 조정 어려움.
### ● 필요 조치
- 의미 있는 지엽적 시나리오 선별하여 적용 예정.
- 부산대 정재준 교수 등 전문가 기술 협력 필요.
- MARS 코드의 현실적 제약 (동적 조작 등) 고려 필요.
### ● 모델/시나리오 방향

- 전체 시스템 단일 제어보다 컴포넌트별 제어 모듈 협력 방식 권장.
- SMART100 기준, 단순화된 구조로 시작.
- Observation/Action 정의가 명확한 소규모 시나리오 우선 (예: 루프 제어, 출력증감발, 비정상 운전 등).

#### 고속 해석 시뮬레이터 개발 TF 구성
자율 운전을 위해서는 실시간으로 해석에 사용되는 변수를 제어(값 변경)할 수 있어야 함. 그러나 현재 MARS의 경우 **실시간 해석 변수 제어 불가능한** 상태임.

MARS 코드 해석 흐름
- 해석에 필요한 각 카드를 작성 (Time 별 계통 컨디션 까지 미리 작성하여 입력함)
- 해석 시작 시 모든 데이터 입력 (한번 해석이 진행되면 별도의 조작 어려움)
- 결과를 모니터링 하는 방식
