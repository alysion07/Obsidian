## Q&A

1. **Pix4Dengine SDK를 사용하지 않고PIX4D Cloud API를 사용하여 개발 된 이유**
	 - 초기 기획 단계에서 코드를 받을 수도 있다는 가능성이 있었으나, 추후에는 불가하다는 답변 
	 - 최대한 빠르게 적용할 수 있도록 Cloud API 사용하여 개발됨

2. **개발 및 운영에 필요한 Cloud PGP 확보 방법 (On-demand 요금?)**
	1. Credit의 존재를 알고 있었는지? : 회의중 언급한 내용(크래딧 아껴야한다는..)으로 보아 직전 전무님과 통화를 통해 알게 되었을 것으로 추측

3. **소스 코드 관련하여**
	- 개발 기간: 25.01.10 ~ 25.07.09
    - 주요 개발 언어:React Framework NEXT.JS & Vercel 활용하여 개발됨
    - 개발 항목 중 아직 완료 되지 않은 기능
	    - 데이터셋의 썸네일 출력 기능은 현재 개발중
	    - 프로젝트 공유 기능 (Read only / Editable 설정 가능) 
    - 현재 기준으로 코드 인계하여 우리 측에서 검토 가능한지
	    - Workflow 별 구분해서 기획했기 때문에 인계 후 온보딩도 문제 없을것으로 예상

4. **pix4d Bucket을 사용하지 않고 Cloud Viewer만 단독으로 사용 가능한 형태인지?** 
	1. viewer 사용을 위해서는 Cloud API를 연동해야만 하는 상황
	2. pix4d Cloud 스토리지를 사용해만 가시화 가능('dataset-id'가 API 전달 인자로 사용됨)
	3. 결국 Credit을 소모하는 형태로 서비스해야 하는 형태
	
5. **프로젝트 관리 기능 관련** 
	- **어떤 방식으로 아웃풋 관리를 하려고 했는지?**
	-  **프로세싱 이후, 결과물 (Mesh, DSM, DTM, Ortho, pcl)을 별도로 가져올 수 있는지**
		- 기존 API에서 제공하는 기능 대비, 특별히 개발된 기능은 없을것으로 예상 
		- API를 그대로 사용했을것으로 예상 

6. **Local PGP의 소모**
	1. 보다 플랫폼을 활용한 Local PGP 소모 계획 중(KCLOUD 활용)
	2. 백엔드 기반 프로세싱 서비스만 제공

3. Pix4d Capture에서 들어오는 데이터 SDK구동 테스트 필요
	1. 국내 GPS 기반으로 촬영된 데이터 처리 결과 테스트
